Symbol Identification Code (SIDC)

A SIDC identifies a military symbol; this project follows APP-6D-aligned data (via milstd app6d in the builder) and the same numeric SIDC layout the server uses. The server stores symbols as a hyphenated SIDC string (10 segments). Entity YAML uses a structured sidc_template with human-readable enum fields; the server compiles that into the canonical hyphenated form. The map and SIDC builder render with milsymbol in APP-6 mode (standard: 'APP6'), using a non-hyphenated SIDC — dashes are stripped only at render time.

Hyphenated segment layout (server)

Segments are joined with -. Each segment length is fixed; digits only.

#FieldLengthYAML / meaning
1version2sidc_template.version (e.g. 10)
2context1context enum → digit
3standard identity1standard_identity enum → digit
4symbol set2symbol_set enum → two-digit code
5status1status enum → digit
6HQTFD1hqtfd — Hqtfd enum in Rust (e.g. not_applicable) or legacy "0"…"7" (see table below)
7amplifier2amplifier — Amplifier enum (e.g. unspecified, battalion_squadron) or legacy two-digit string; must match symbol_set
8entity6entity — code_###### (e.g. code_120204) or legacy "120204"
9modifier 12modifier_one — code_## or legacy "00"…"99"
10modifier 22modifier_two — same as modifier 1

amplifier, entity, modifiers

amplifier is two digits (amplifier + descriptor in the standard). Templates often use 00. entity is six digits identifying the specific symbol within the symbol set. Use the interactive picker below to choose valid combinations, then transcribe into YAML. modifier_one and modifier_two are two digits each (often 00).
